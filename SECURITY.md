# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please email: security@XpertRF.com

**Please do not create a public GitHub issue.**

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Security Measures

- All dependencies are regularly updated
- Code is scanned for vulnerabilities
- HTTPS is enforced in production
- Content Security Policy (CSP) is implemented
- XSS protection is enabled
- Secure headers are configured

## Best Practices

- Never commit sensitive data (API keys, passwords)
- Use environment variables for configuration
- Sanitize user inputs
- Validate all data on both client and server
- Keep dependencies up to date
